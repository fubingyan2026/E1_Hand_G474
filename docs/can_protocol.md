# CAN FD 电机控制协议

| 项目 | 内容 |
|------|------|
| 版本 | V1.0 |
| 作者 | maximillian |
| 日期 | 2026-07-09 |
| 概述 | 灵巧手控制板与上位机 CAN FD 通信协议，一帧控制全部 9 个关节电机，周期性自动反馈状态 |


## 1. 物理层

| 项目 | 值 |
|------|-----|
| 接口 | CAN FD (FDCAN1, PA11 RX / PA12 TX) |
| 帧格式 | 标准帧 (11-bit ID) + FD 数据段 |
| 字节序 | 小端 (Intel, LSB first) |
| 模式 | FDCAN_FRAME_FD_NO_BRS (FD 无比特率切换) |

## 2. CAN FD DLC 编码

CAN FD 帧长度仅支持以下 DLC 值：

| DLC | 字节数 |
|-----|--------|
| 0-8 | 0-8 |
| 9 | 12 |
| 10 | 16 |
| 11 | 20 |
| 12 | 24 |
| 13 | 32 |
| 14 | 48 |
| 15 | 64 |

## 3. 帧定义

### 3.1 控制帧 (Host → Device)

| 项目 | 值 |
|------|-----|
| CAN ID | `0x100` |
| DLC | 15 (64 字节) |
| 有效数据 | 55 字节 + 9 字节保留 |

| 偏移 | 字节 | 字段 | 类型 | 格式 | 说明 |
|------|------|------|------|------|------|
| 0 | 1 | ctrl | uint8_t | — | bit0=使能全部, bit1=去使能全部 |
| 1-18 | 18 | pos_ref[9] | int16_t[9] | Q7 LE | 目标角度, 值 = 角度(°) × 128 |
| 19-36 | 18 | spd_ref[9] | int16_t[9] | Q15 LE | 目标速度, 0~32767 → 0%~100% (50krpm) |
| 37-54 | 18 | cur_ref[9] | int16_t[9] | Q15 LE | 电流限制, 0~32767 → 0~5.625A |
| 55-63 | 9 | reserved | — | 0 | 保留, 填充 0 |

**示例** — 设置全部电机到 90°、限速 50%、限流 2A:

| 字段 | 十六进制 | 十进制 | 说明 |
|------|---------|--------|------|
| ctrl | `0x01` | 1 | 使能全部 |
| pos_ref[0..8] | `0x2D00` | 11520 | 90° × 128 |
| spd_ref[0..8] | `0x4000` | 16384 | 50% × 32768 |
| cur_ref[0..8] | `0x1C71` | 7281 | 2A / 5.625 × 32768 |

### 3.2 高频反馈帧 (Device → Host)

| 项目 | 值 |
|------|-----|
| CAN ID | `0x101` |
| 周期 | 100ms |
| DLC | 15 (64 字节) |

| 偏移 | 字节 | 字段 | 类型 | 格式 | 说明 |
|------|------|------|------|------|------|
| 0 | 1 | global_state | uint8_t | — | 行为状态 (见 §4.1) |
| 1-9 | 9 | fsm_state[9] | uint8_t[9] | — | 每电机硬件状态 (见 §4.2) |
| 10-27 | 18 | angle_fb[9] | int16_t[9] | Q7 LE | 当前角度, 值 / 128 = 角度(°) |
| 28-45 | 18 | speed_fb[9] | int16_t[9] | Q15 LE | 当前转速 (见 §6) |
| 46-63 | 18 | q_cur[9] | int16_t[9] | Q15 LE | Q轴电流 (见 §6) |

### 3.3 低频状态帧 (Device → Host)

| 项目 | 值 |
|------|-----|
| CAN ID | `0x102` |
| 周期 | 500ms |
| DLC | 14 (48 字节) |
| 有效数据 | 46 字节 + 2 字节保留 |

| 偏移 | 字节 | 字段 | 类型 | 格式 | 说明 |
|------|------|------|------|------|------|
| 0 | 1 | global_state | uint8_t | — | 行为状态 (见 §4.1) |
| 1-9 | 9 | fsm_state[9] | uint8_t[9] | — | 每电机硬件状态 (见 §4.2) |
| 10-18 | 9 | err_code[9] | uint8_t[9] | — | 每电机故障码 (见 §4.3) |
| 19-27 | 9 | temp[9] | int8_t[9] | — | MCU 温度 (见 §6) |
| 28-45 | 18 | vbus[9] | int16_t[9] | LE | 母线电压 (见 §6) |
| 46-47 | 2 | reserved | — | 0 | 保留 |

## 4. 枚举值参考

### 4.1 global_state (行为状态)

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | INIT | 初始, 等待首次反馈 |
| 1 | OFFLINE | 离线, 通信超时 |
| 2 | IDLE | 在线, 已就绪 (mcReady) |
| 3 | ENABLING | 使能中, 校准进行中 |
| 4 | RUNNING | 正常运行 (mcRun) |
| 5 | FAULT | 故障, 已自动去使能 |

### 4.2 fsm_state (电机端硬件状态)

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | mcReady | 就绪 |
| 5 | mcAlign | 预定位/校准中 |
| 7 | mcRun | 运行中 |
| 9 | mcFault | 故障 |

### 4.3 err_code (电机故障码)

| 值 | 说明 |
|----|------|
| 0 | 无故障 |
| 1 | 硬件过流 |
| 2 | 软件过流 |
| 3 | 硬件过压 |
| 4 | 软件过压 |
| 5 | 硬件欠压 |
| 6 | 软件欠压 |
| 7 | 缺相 |
| 8 | 电机堵转 |
| 9 | 过温保护 |
| 10 | 串口通信丢失 |
| 11 | 内部自检故障 |

## 5. 电机索引

| 索引 | 组 | UART | 电机 ID |
|------|-----|------|---------|
| 0-4 | A | USART2 | 1-5 |
| 5-8 | B | USART3 | 1-4 |

## 6. 速率换算

| 字段 | 数据类型 | 还原公式 | 单位 |
|------|---------|---------|------|
| pos_ref / angle_fb | Q7 (int16_t) | deg = value / 128 | ° (度) |
| spd_ref / speed_fb | Q15 (int16_t) | krpm = value × 50 / 32768 | krpm |
| cur_ref / q_cur | Q15 (int16_t) | A = value × 5.625 / 32768 | A (安培) |
| temp | int8_t | °C = value − 50 | °C |
| vbus | int16_t | V = value / 128 | V (伏特) |
